🏢 Claude Code と Claude Cowork の違い

「開発者向けの黒い画面」か、「誰でも使える便利アプリ」か

💻 Claude Code
凄腕のハッカー・職人

特徴: ターミナル（黒い画面）やVS Codeなどのエディタで動く開発者向けツール。コードの生成、ファイルの作成・書き換え、テスト実行などを自律的に行います。

得意技: システム開発、エラーの修正、環境構築など「作る」作業。

👩‍💼 Claude Cowork
オフィスの超優秀な秘書

特徴: Claudeのデスクトップアプリに搭載された、チャット画面（GUI）で動く機能。非エンジニアでもマウス操作と自然言語で簡単に使えます。

得意技: ファイル整理、メールの自動分類、データ集計など「事務・整理」作業。

🍎 Mac：セットアップ手順

3ステップで完了します

⚠️ 前提: Claudeの有料プラン（Pro, Maxなど）の契約が必要です。

Step 1：ターミナルを開く

Finder → アプリケーション → ユーティリティ → ターミナル.app
または、Cmd + Space で Spotlight を開き「ターミナル」と入力してEnter。

Step 2：インストールコマンドを実行

以下を丸ごとコピペしてEnterを押すだけ：

curl -fsSL https://claude.ai/install.sh | bash

Step 3：起動する

作業したいフォルダに移動して claude と入力しEnter。初回のみブラウザが開いてログイン認証が行われます。

cd ~/Desktop/my-project
claude

🔧 Mac：うまくいかない時は？

よくあるトラブルと解決法

❌ 「command not found: curl」と出る

  • 原因: Xcode Command Line Tools が未インストール
  • 解決: 先に以下を実行してからやり直す
xcode-select --install

❌ 「Node.js is required」と出る

  • 原因: Node.js 18以上が入っていない
  • 確認: node -v でバージョンを確認
  • 解決: nodejs.org からLTS版をインストール

❌ 「Permission denied」と出る

  • 原因: インストール先への書き込み権限がない
  • 解決: コマンドの先頭に sudo を付けて再実行
sudo curl -fsSL https://claude.ai/install.sh | bash

❌ インストール後「command not found: claude」

  • 原因: PATHが通っていない（シェルが新しい設定を読めていない）
  • 解決: ターミナルを一度閉じて開き直す。または以下を実行：
source ~/.zshrc

💡 会社のPCで通信エラーが出る場合は、社内プロキシやファイアウォールが原因の可能性があります。情シス部門に相談してみてください。

🔧 Windows：うまくいかない時は？

よくあるトラブルと解決法

❌ 「スクリプトの実行が無効」と出る

  • 原因: PowerShellの実行ポリシーでスクリプトがブロックされている
  • 解決: PowerShellを管理者として実行し、以下を先に実行：
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

❌ 「Node.js is required」と出る

  • 原因: Node.js 18以上が入っていない
  • 確認: node -v でバージョンを確認
  • 解決: nodejs.org からLTS版をインストール

❌ 「git が認識されません」と出る

  • 原因: Git for Windowsが未インストール
  • 解決: gitforwindows.org からインストール後、PowerShellを再起動

❌ インストール後「claude は認識されません」

  • 原因: PATHが通っていない
  • 解決: PowerShellを一度閉じて開き直す。それでもダメならPCを再起動

💡 会社のPCで通信エラーが出る場合は、社内プロキシやファイアウォールが原因の可能性があります。情シス部門に相談してみてください。

📁 ディレクトリ（フォルダ）の考え方

Claude Codeは「呼ばれた場所」で働きます。設定の有効範囲を理解しましょう。

🏢 プロジェクト専用
その部署だけのルール

場所: 作業フォルダ内の .claude/ フォルダ

特徴: 「このAプロジェクトの時だけはこのルールで動いてね」という設定を保存。チームメンバーに共有する際にも便利です。

🌍 個人専用
あなた自身のマイルール

場所: PCの大元にある ~/.claude/ フォルダ

特徴: 「どんなプロジェクトでも、私と作業する時は必ず日本語で話してね」といった、全プロジェクト共通の基本設定を置きます。

🧠 コンテキストとは？（超重要）

AIが効率よく働くための「作業机の広さ」のことです。

📝 コンテキスト = AIの短期記憶（ワーキングメモリ）

会話履歴、読み込んだファイル、実行したコマンドなどをすべて「コンテキスト」という記憶領域に保持しながら作業します。

⚠️ 問題（コンテキストの圧迫）

作業机に書類を広げすぎると → 「指示を忘れる」「動作が遅くなる」「適当な嘘をつく（ハルシネーション）」

💡 対策

/clear … 話題が変わったら机の書類を全部捨てて真っさらにする

/compact … 散らかった書類を「要約」して小さなメモ帳に圧縮する

📖 CLAUDE.md とは？

配属初日の新入社員（Claude）に渡す「会社の就業規則・理念」です。

📋 プロジェクトのフォルダ直下に置く CLAUDE.md というテキストファイル。
Claudeは作業開始前に、必ず毎回このファイルを読み込みます。

✍️ 書くべきこと

  • このプロジェクトの目的（What）
  • 絶対に守ってほしいコーディングルールやデザイン規約（How）
  • やってはいけない禁止事項

⚠️ 注意: 毎回読み込まれるため、長すぎるとコンテキストを最初から圧迫します。
本当に重要な基本ルールだけを短く書く（レス・イズ・モア）のがコツです。

📋 Skills（スキル）とは？

作業机の引き出しに入っている「定型業務の手順書（マニュアル）」です。

📚 よくやる作業（議事録の作成、コードレビュー、リサーチなど）の手順をまとめたファイル。

✨ 特徴（魔法の引き出し）

  • 人間が指示しなくても、Claudeが依頼内容を見て自動的に適切なマニュアルを取り出して実行
  • 普段は引き出しにしまってあるので、コンテキストを圧迫しない

🔑 作り方のコツ

ファイル上部の description（説明）に「このマニュアルは〇〇の時に使います」と明確に書いておくことが最重要ポイント。

これにより、Claudeが正しいタイミングで自動選択できるようになります。
