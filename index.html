🏢 Claude Code と Claude Cowork の違い

「開発者向けの黒い画面」か、「誰でも使える便利アプリ」か

💻 Claude Code
凄腕のハッカー・職人

特徴: ターミナル（黒い画面）やVS Codeなどのエディタで動く開発者向けツール。コードの生成、ファイルの作成・書き換え、テスト実行などを自律的に行います。

得意技: システム開発、エラーの修正、環境構築など「作る」作業。

👩‍💼 Claude Cowork
オフィスの超優秀な秘書

特徴: Claudeのデスクトップアプリに搭載された、チャット画面（GUI）で動く機能。非エンジニアでもマウス操作と自然言語で簡単に使えます。

得意技: ファイル整理、メールの自動分類、データ集計など「事務・整理」作業。

🔧 Mac：うまくいかない時は？

よくあるトラブルと解決法

❌ 「command not found: curl」と出る

  • 原因: Xcode Command Line Tools が未インストール
  • 解決: 先に以下を実行してからやり直す
xcode-select --install

❌ 「Node.js is required」と出る

  • 原因: Node.js 18以上が入っていない
  • 確認: node -v でバージョンを確認
  • 解決: nodejs.org からLTS版をインストール

❌ 「Permission denied」と出る

  • 原因: インストール先への書き込み権限がない
  • 解決: コマンドの先頭に sudo を付けて再実行
sudo curl -fsSL https://claude.ai/install.sh | bash

❌ インストール後「command not found: claude」

  • 原因: PATHが通っていない（シェルが新しい設定を読めていない）
  • 解決: ターミナルを一度閉じて開き直す。または以下を実行：
source ~/.zshrc

💡 会社のPCで通信エラーが出る場合は、社内プロキシやファイアウォールが原因の可能性があります。情シス部門に相談してみてください。

🔧 Windows：うまくいかない時は？

よくあるトラブルと解決法

❌ 「スクリプトの実行が無効」と出る

  • 原因: PowerShellの実行ポリシーでスクリプトがブロックされている
  • 解決: PowerShellを管理者として実行し、以下を先に実行：
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

❌ 「Node.js is required」と出る

  • 原因: Node.js 18以上が入っていない
  • 確認: node -v でバージョンを確認
  • 解決: nodejs.org からLTS版をインストール

❌ 「git が認識されません」と出る

  • 原因: Git for Windowsが未インストール
  • 解決: gitforwindows.org からインストール後、PowerShellを再起動

❌ インストール後「claude は認識されません」

  • 原因: PATHが通っていない
  • 解決: PowerShellを一度閉じて開き直す。それでもダメならPCを再起動

💡 会社のPCで通信エラーが出る場合は、社内プロキシやファイアウォールが原因の可能性があります。情シス部門に相談してみてください。

🛠️ NotebookLM Skill セットアップ

3つのコマンドでインストール完了。あとはClaude Codeが自動でやってくれます。

⚠️ 前提: Python 3.8以上 ＋ Google アカウント ＋ Claude Code が必要です。

Step 1：Skillをインストール

以下の3行をターミナルにコピペして実行：

mkdir -p ~/.claude/skills
cd ~/.claude/skills
git clone https://github.com/PleasePrompto/notebooklm-skill notebooklm

Step 2：Claude Codeで認証設定

Claude Codeを起動して以下を入力。Chromeが開くのでGoogleアカウントでログイン：

Set up NotebookLM authentication

※ 認証は初回のみ。以降はセッション間で永続化されます。

Step 3：ノートブックを登録

NotebookLMで文書をアップロード後、そのURLをClaude Codeに教える：

Add https://notebooklm.google.com/notebook/xxxxx to my NotebookLM library

💡 依存パッケージ（patchright等）は初回実行時にスキルフォルダ内の仮想環境に自動インストールされます。グローバル環境は汚れません。

🔌 MCP（Model Context Protocol）とは？

AIと外部サービスをつなぐ「共通規格のUSBポート」です

💡 MCP = AIモデルが外部のツールやデータに安全にアクセスするための共通プロトコル（通信規格）。
従来のAPI連携が「サービスごとに専用ケーブルを作る」のに対し、MCPは「USBのように1つの規格で何でもつながる」仕組みです。

🚫 従来のAPI連携
サービスごとに専用開発が必要

  • サービスAのAPI仕様を調査 → 専用コードを書く
  • サービスBは別の認証方式 → また別のコードを書く
  • 仕様変更のたびに修正が必要
  • 結果: 連携先が増えるほど開発・保守コストが膨大に

✅ MCP連携
共通規格で「つなぐだけ」

  • MCPサーバーを設定ファイルに追加するだけ
  • 認証・データ形式・エラー処理が標準化済み
  • Claude Codeが自動的に使えるツールを認識
  • 結果: エンジニアでなくても外部連携が可能に

📋 Skills / スキル

定型業務のワークフローを手順書化し、高い再現性で実行

スキルとは？

  • 繰り返し使う手順を「スキル」として登録
  • 「/スキル名」で呼び出すだけで実行
  • 毎回同じ品質のアウトプットを保証
  • チームで共有して業務を標準化

活用例

  • 「/議事録作成」→ テンプレートで議事録を自動生成
  • 「/週報作成」→ フォーマット通りに週報を出力
  • 「/メール返信」→ 状況に合わせた返信文を生成

📌 Claude Codeのスキルと概念は同じ。Web版でも「/」コマンドで定型業務を自動化できる

🔌 Plugins / プラグイン

スキルと外部ツール接続をまとめたパッケージ機能

プラグインとは？

  • スキル＋外部ツール連携をセットにしたパッケージ
  • インストールするだけで特定の業務フローが使える
  • サードパーティ製プラグインも利用可能

例

  • Notion連携プラグイン → Notionへの直接書き込みが可能に
  • Slack連携プラグイン → 特定チャンネルへの投稿を自動化
  • 業務特化プラグイン → 業界別ワークフローを即利用

📌 スキルの「手順書」にツール接続を加えた、より完成度の高い自動化パッケージ
